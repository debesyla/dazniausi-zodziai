# Project workflow

Track work in GitHub Issues. For implementation work that changes project files:

1. Work on a focused `codex/` branch tied to the issue or task.
2. Run the relevant verification before handoff.
3. Create a focused commit, push the branch, and open a pull request for review.
4. Report the commit and pull-request links in the handoff.

Do not commit or include unrelated pre-existing worktree changes.
